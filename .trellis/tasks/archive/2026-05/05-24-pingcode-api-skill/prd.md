# Create PingCode API skill

## Goal

Build a Codex skill for operating PingCode through the official REST API. The skill should help an AI agent discover PingCode IDs, create and update project work items, create product ideas, and manage common project/product records using OAuth2 client credentials.

## What I already know

* The user wants a new project whose output is a reusable AI skill for PingCode.
* The preferred auth flow is `client_credentials`.
* Core user scenarios are updating a work item status, creating work items, creating stories/ideas, and covering product management plus project management APIs.
* The repository is currently a new shell with Trellis and agent configuration only.
* Official docs are hosted at `https://open.pingcode.com/` and expose apidoc data through `https://open.pingcode.com/api_data.json`.

## Requirements

* Provide a root `SKILL.md` with concise invocation guidance and progressive links to references.
* Provide executable Python CLI scripts that read credentials from environment variables, never hard-code secrets.
* Use PingCode public cloud base URL `https://open.pingcode.com` by default, with `PINGCODE_BASE_URL` override for private deployments.
* Support OAuth2 client credentials: `PINGCODE_CLIENT_ID`, `PINGCODE_CLIENT_SECRET`, optional token cache.
* Support common project management operations:
  * list projects
  * list work item types, states, priorities, sprints, boards, entries, swimlanes
  * list/create/update work items
  * update a work item status via `state_id`
* Support product management operations:
  * list products
  * list idea states, priorities, plans, suites
  * list/create/update product ideas
* Provide a generic request command for endpoints not wrapped by convenience subcommands.
* Provide references with endpoint summaries and AI workflow guidance for resolving names into IDs before mutation.
* Provide tests for payload construction, auth behavior, request handling, and CLI parser coverage without live network calls.

## Acceptance Criteria

* [ ] `python3 scripts/pingcode.py --help` shows usable subcommands.
* [ ] CLI can run in dry-run mode for create/update commands and print method, path, query, and JSON body.
* [ ] Unit tests pass without external network access.
* [ ] Skill documentation explains environment variables, safety checks, and when to load references.
* [ ] No credentials are committed.

## Out of Scope

* Running live write operations against the user's PingCode tenant during implementation.
* Full coverage of every PingCode REST endpoint.
* OAuth authorization-code user tokens.
* Building an MCP server.

## Technical Notes

* Official API docs are generated by apidoc; `main.js` loads `api_project.js` and `api_data.js`.
* `api_project.js` reports generator timestamp `2026-04-30T10:32:01.154Z`.
* PingCode docs state public cloud `rest_api_root` is `https://open.pingcode.com`.
* Rate limit is 200 requests per minute per identity; 429 responses include `x-pc-retry-after`.
