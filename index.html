面向项目管理的高频能力

把 PingCode 的常用操作包装成 Agent 可以稳定调用的 skill，而不是零散提示词。

01

查询工作项

按当前用户、项目、迭代查询任务、故事和缺陷，也支持显式查询全部范围。

02

创建和更新

创建故事、任务、缺陷，在故事下添加子工作项，并更新工作项状态。

03

产品需求

查询产品和产品需求，覆盖项目管理与产品管理两类核心场景。

04

上下文缓存

缓存当前用户、项目、迭代和状态字典，减少重复选择和无效 API 调用。

05

安全默认值

写操作建议先 dry-run；不会把 client secret、access token 写入仓库。

06

统一 CLI

通过单个脚本调用 PingCode REST API，输出 JSON，方便 Agent 解析和追踪。

三步启用

部署到本机 Agent 后，日常操作就可以交给自然语言入口。

安装到 Agent

一条 npx pingcode-skill@latest 检测当前用户已有 Agent，只安装到对应个人 skills 目录。

配置 PingCode 凭证

设置 PINGCODE_CLIENT_ID 和 PINGCODE_CLIENT_SECRET。

初始化工作区

运行 pingcode-ctx 选择当前用户、项目和迭代。
